    Отчет по лабораторной работе №4¶

    Дисциплина: Web-программирование
    Тема: Реализация клиентской части приложения средствами Vue.js
    Вариант: Система управления гостиницей

    Выполнил: Абакунов Кирилл, группа K3340

    1. Описание варианта¶

    Реализована клиентская часть системы управления гостиницей на Vue.js 3 с использованием Vuetify 3. Приложение представляет собой SPA (Single Page Application), которое взаимодействует с REST API бэкенда на Django REST Framework, реализованного в лабораторной работе №3.

    Приложение обеспечивает полноценное управление гостиницей через современный веб-интерфейс с поддержкой всех необходимых операций.

    2. Технологии¶

    Технология Версия Назначение
    Vue.js 3.4 Основной фреймворк
    Vuetify 3.4 UI библиотека
    Pinia 2.1 State management
    Vue Router 4.2 Маршрутизация
    Axios 1.6 HTTP клиент
    Vite 5.0 Сборщик

    4. Архитектура приложения¶

    4.1. API клиент (Axios)¶

    Для взаимодействия с бэкендом используется Axios с настроенным базовым URL и interceptors для автоматической подстановки токена авторизации.

    Модули API:

    • authAPI — авторизация, регистрация, профиль, смена пароля
    • roomsAPI — CRUD номеров, получение свободных номеров, история номеров
    • guestsAPI — CRUD гостей, заселение, выселение, история проживания
    • staffAPI — CRUD персонала, найм, увольнение
    • schedulesAPI — CRUD расписаний уборки по этажам и дням недели
    • reportsAPI — получение квартальных отчётов о доходах

    Настройка interceptors:

    • Автоматическое добавление токена авторизации в заголовки запросов
    • Обработка ошибок 401 (неавторизован) с автоматическим редиректом на страницу входа
    • Обработка ошибок сети и сервера

    4.2. Управление состоянием (Pinia)¶

    Приложение использует Pinia для централизованного управления состоянием.

    Stores:

    • auth — состояние авторизации (user, token, loading, error), методы login, register, logout, updateUser, changePassword
    • rooms — состояние номеров (список, текущий номер, фильтры), методы загрузки и CRUD операций
    • guests — состояние гостей (список, история), методы заселения/выселения
    • staff — состояние персонала (список), методы найма/увольнения
    • schedules — состояние расписаний уборки

    4.3. Маршрутизация (Vue Router)¶

    Приложение использует Vue Router для навигации между страницами с защитой маршрутов.

    Основные маршруты:

    • / — главная страница (статистика)
    • /login — страница входа
    • /register — страница регистрации
    • /profile — профиль пользователя (требует авторизации)
    • /rooms — управление номерами (требует авторизации)
    • /guests — управление гостями (требует авторизации)
    • /staff — управление персоналом (требует авторизации)
    • /schedules — расписания уборки (требует авторизации)
    • /reports — квартальные отчёты (требует авторизации)

    Защита маршрутов: Используются navigation guards для проверки авторизации перед доступом к защищённым страницам.

    5. Основные возможности¶

    5.1. Авторизация и регистрация¶

    Страница входа (/login):

    • Форма с полями: имя пользователя, пароль
    • Валидация полей на клиенте
    • Обработка ошибок авторизации
    • Автоматическое сохранение токена в localStorage
    • Редирект на главную страницу после успешного входа

    Страница регистрации (/register):

    • Форма с полями: имя пользователя, email, пароль, подтверждение пароля
    • Валидация: минимум 3 символа для имени, формат email, минимум 8 символов для пароля
    • Проверка совпадения паролей
    • Автоматический вход после успешной регистрации

    Страница профиля (/profile):

    • Вкладка "Информация": редактирование имени пользователя, email, имени, фамилии
    • Вкладка "Смена пароля": изменение пароля с подтверждением текущего
    • Валидация всех полей
    • Отображение успешных сообщений об обновлении

    5.2. Управление номерами¶

    Страница номеров (/rooms):

    • Список номеров: отображение всех номеров в виде карточек с информацией (номер, этаж, тип, статус, цена)
    • Фильтрация: по этажу, типу, статусу (свободен/занят)
    • Создание номера: диалоговое окно с формой (номер, этаж, тип, цена за сутки, описание)
    • Редактирование номера: изменение всех полей существующего номера
    • Удаление номера: подтверждение перед удалением
    • Просмотр истории: история заселений для конкретного номера
    • Свободные номера: фильтр и отображение только свободных номеров

    5.3. Управление гостями¶

    Страница гостей (/guests):

    • Список гостей: таблица с информацией о всех гостях (ФИО, email, номер, даты заселения/выселения, статус)
    • Заселение: форма выбора номера, даты заезда, данных гостя (ФИО, email, телефон)
    • Выселение: выбор гостя и даты выселения, автоматический расчёт стоимости
    • История проживания: просмотр истории всех заселений конкретного гостя
    • Редактирование: изменение данных гостя
    • Удаление: удаление записи о госте

    5.4. Управление персоналом¶

    Страница персонала (/staff):

    • Список служащих: таблица с информацией (ФИО, должность, дата найма, статус)
    • Найм: форма добавления нового служащего (ФИО, должность, дата найма, контакты)
    • Увольнение: выбор служащего и даты увольнения
    • Редактирование: изменение данных служащего
    • Просмотр: детальная информация о служащем

    5.5. Расписания уборки¶

    Страница расписаний (/schedules):

    • Календарь уборки: отображение расписания по этажам и дням недели
    • Создание расписания: выбор этажа, дня недели, служащего, времени
    • Редактирование: изменение существующего расписания
    • Удаление: удаление записи расписания
    • Фильтрация: по этажу, дню недели, служащему

    5.6. Отчёты¶

    Страница отчётов (/reports):

    • Выбор периода: выбор года и квартала (1-4)
    • Общая статистика: карточки с ключевыми показателями (общий доход, количество заселений, средний доход)
    • Статистика по этажам: доходы по каждому этажу
    • Детальная таблица: доходы по каждому номеру с возможностью сортировки
    • Визуализация: график топ-10 номеров по доходам
    • Экспорт: возможность экспорта отчёта

    6. Настройка CORS¶

    Для работы фронтенда с бэкендом необходимо настроить CORS на стороне Django сервера.

    Настройка в Django (lab3):

    # settings.py
INSTALLED_APPS = [
    ...
    'corsheaders',
]

MIDDLEWARE = [
    ...
    'corsheaders.middleware.CorsMiddleware',
    'django.middleware.common.CommonMiddleware',
    ...
]

# Разрешить запросы с фронтенда
CORS_ALLOWED_ORIGINS = [
    "http://localhost:5173",  # Vite dev server
    "http://127.0.0.1:5173",
]

# Или разрешить все (только для разработки!)
# CORS_ALLOW_ALL_ORIGINS = True

CORS_ALLOW_CREDENTIALS = True

    Настройка Axios: В клиенте настроен базовый URL и автоматическая отправка credentials с запросами.

    7. Запуск проекта¶

    # Установка зависимостей
npm install

# Запуск в режиме разработки
npm run dev

# Сборка для продакшена
npm run build

    Приложение будет доступно по адресу http://localhost:5173 (или другому порту, указанному Vite).

    8. Связь с бэкендом¶

    Приложение подключается к бэкенду из lab3, работающему на http://localhost:8000. Убедитесь, что бэкенд запущен перед использованием фронтенда.

    Базовый URL API: http://localhost:8000/api/

    Основные эндпоинты:

    • /api/auth/ — аутентификация (login, register, profile)
    • /api/rooms/ — управление номерами
    • /api/guests/ — управление гостями
    • /api/staff/ — управление персоналом
    • /api/schedules/ — расписания уборки
    • /api/reports/ — квартальные отчёты

    Формат данных: Все запросы и ответы используют JSON формат. Токен авторизации передаётся в заголовке Authorization: Token <token>.

    9. UI/UX особенности¶

    Приложение использует Material Design через библиотеку Vuetify 3, что обеспечивает:

    • Адаптивный дизайн: корректное отображение на всех устройствах
    • Тёмная/светлая тема: переключение темы оформления
    • Компоненты: использование готовых компонентов Vuetify (таблицы, формы, диалоги, карточки)
    • Валидация форм: клиентская валидация с отображением ошибок
    • Уведомления: toast-уведомления об успешных операциях и ошибках
    • Загрузка: индикаторы загрузки при выполнении запросов
    • Навигация: удобное меню навигации с активными ссылками

    ✅ Авторизация и регистрация с использованием токенов
    ✅ Управление номерами (CRUD операции)

    ✅ Управление персоналом (найм, увольнение, просмотр)
    ✅ Расписания уборки по этажам и дням недели
    ✅ Квартальные отчёты о доходах гостиницы
    ✅ Настроен CORS для взаимодействия с бэкендом

    Приложение имеет современный пользовательский интерфейс на базе Material Design (Vuetify), реактивное управление состоянием через Pinia, маршрутизацию через Vue Router и полностью интегрировано с REST API бэкенда.
